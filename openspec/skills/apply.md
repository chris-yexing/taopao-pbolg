# openspec-apply

当用户想要实施某个 OpenSpec 变更、开始写代码、或继续执行任务时，按以下步骤执行。

## 目标

读取指定变更的所有制品，按 tasks.md 中的任务列表逐步实施，直到完成或遇到阻碍。

## 执行步骤

### 1. 确定目标变更

如果用户指定了变更名称，直接使用。否则：
- 列出 `openspec/changes/` 下的所有目录（不含 `archive/`）
- 如果只有一个活跃变更，自动选择并告知用户
- 如果有多个，询问用户选择哪个

告知用户："正在使用变更：<name>"

### 2. 读取全部上下文

按顺序读取以下文件：
- `openspec/changes/<name>/proposal.md`
- `openspec/changes/<name>/design.md`
- `openspec/changes/<name>/specs/` 下所有 spec.md
- `openspec/changes/<name>/tasks.md`

**如果 tasks.md 不存在，停止并提示用户先完成制品创建（运行 propose 技能）。**

### 3. 统计任务进度

从 tasks.md 统计：
- 总任务数
- 已完成（`- [x]`）数量
- 待完成（`- [ ]`）数量

展示进度：`N/M 任务完成`

如果全部已完成，提示用户考虑归档此变更。

### 4. 逐任务实施

对每个待完成任务：

1. 显示当前任务："正在实施 [N/M]：<任务描述>"
2. 参照对应 spec 的验收条件实施代码变更
3. 保持改动最小化，只做任务要求的内容
4. 完成后立即将 tasks.md 中该任务标记为 `- [x]`
5. 继续下一个任务

**遇到以下情况时暂停并等待用户指示：**
- 任务描述不清晰
- 实施中发现设计问题需要更新制品
- 遇到错误或阻碍

### 5. 完成或暂停后输出状态

**全部完成时：**
```
实施完成：<name>
进度：M/M 任务全部完成 ✓

本次完成：
- [x] 任务1
- [x] 任务2

可以归档此变更了。
```

**暂停时：**
```
实施暂停：<name>
进度：N/M 任务完成

暂停原因：<描述>

请指示如何继续。
```

## 注意事项

- 始终先读取全部上下文再开始实施
- 任务完成后立即更新 tasks.md，不要批量更新
- 如果实施中发现设计问题，暂停并建议更新 design.md 或 specs/，不要强行实施
- 代码改动保持最小化，不做任务范围外的重构或优化
