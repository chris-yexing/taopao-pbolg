## Why

博客当前的亮/暗模式依赖读者手动切换，或跟随系统设置。但系统设置是全局偏好，不能反映当前时刻的实际光照环境。深夜用亮色阅读伤眼，白天用暗色视觉疲劳，读者体验存在明显痛点。

以日出/日落作为切换节点，是最符合人类生理节律的方案：比固定时间（如 18:00）更精确，比系统 dark mode 更有情境感知能力，同时保留手动覆盖的自由。

## What Changes

在现有 PaperMod 主题的基础上，注入一段纯客户端 JavaScript，实现：

1. 在请求地理位置权限前，向读者展示一条说明，告知位置信息仅用于计算日出/日落，不离开浏览器
2. 请求浏览器地理位置权限，获取读者经纬度（L1）
3. 若读者拒绝授权，降级为时区推算近似坐标（L2）；若时区也不可用，降级为系统 `prefers-color-scheme`（L3）
4. 在客户端计算当地日出/日落时间（无需外部 API）
5. 根据当前时刻与日出/日落的对比，自动将主题设为 `light` 或 `dark`
6. 保留 PaperMod 原有手动切换按钮，读者手动切换后当前 session 内不再自动覆盖

## Capabilities

### New Capabilities

- `sunrise-sunset-theme`: 基于读者本地日出/日落时间，在页面加载时自动设置亮/暗主题，含三层降级逻辑
- `geolocation-consent-hint`: 在触发浏览器权限弹窗前，展示友好说明告知读者请求目的

### Modified Capabilities

- `theme-toggle`（PaperMod 内置）：手动切换按钮保留且优先级高于自动逻辑；手动切换后 session 内不再被自动覆盖

## Impact

- **纯客户端**：无需后端、无需外部 API 调用，不影响构建流程和部署
- **非侵入式**：通过 PaperMod 的 `extend_head.html` / `extend_footer.html` 扩展点注入，不修改主题文件
- **隐私透明**：请求权限前主动说明用途，地理位置仅在浏览器内使用，不上报任何服务器
- **渐进增强**：三层降级确保拒绝授权的读者仍能享受基于时区的日出日落切换
- **用户控制**：自动逻辑不剥夺手动控制权，两者共存且手动优先
